## Bug 详情

### BUG-001：POST /posts 缺少必填字段仍返回201

| 字段 | 内容 |
|------|------|
| **所属模块** | Posts |
| **严重程度** | 中 |
| **优先级** | P2 |
| **复现步骤** | `POST /posts`，body: `{"title": "test only"}`（缺少 body 和 userId） |
| **预期结果** | 应返回 `400 Bad Request`，提示缺少必填字段 body 和 userId |
| **实际结果** | 返回 `201 Created`，缺失字段值为 null |
| **状态** | 已确认（API 设计如此） |
| **影响** | 无法验证客户端对必填字段的校验逻辑 |
| **建议** | 对接真实 API 时，需确认服务端是否对必填字段做校验 |

## 总结

本次测试发现的 5 个问题均为 JSONPlaceholder Mock API 的已知设计限制，包括：

- 写操作不持久化（数据不会真正保存到服务端）
- 必填字段不校验（缺少字段仍返回成功）
- 部分字段更新不反映在响应中

这些问题不影响对 REST API 测试方法论和自动化测试框架的验证。在对接真实后端 API 时，上述问题应作为重点验证项。

**建议**：在后续的真实项目测试中，重点关注数据持久化、字段校验、响应数据一致性三个方面。
